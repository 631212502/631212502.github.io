保供电之路，风险防误3原则：慎，谨/细，慢

风险或者重大失误出现的情形

不做安措，不做保命措施，随意接触带电设备乃至高压设备。
完全规避失误和风险是一种伪命题，它只能作为结果存在，不能作为前提去寻找，态度谨小慎微，工作细心仔细，多问多确认，才能尽可能给安全和不背锅保驾护航。而且出事以后也十分考验对内对外的处理能力。说不定机遇就藏在风险当中。

工作经验搜索树

2025年6月下旬的某一天，天渐亮，可我却做了一个噩梦，梦见自己把运行中的设备弄跳闸了，那一瞬间自己的心跳仿佛都停止了，呼吸凝固，脑细胞仿佛在成片死亡，无法接受这样的事实；最终梦醒，可自己却有着仿佛亲身经历般的后怕：

  1. 做任何非监护操作必须拍照或者拍视频留底，工作留痕
  2. 任何疑问必须请示上级/分部或者寻找依据资料支撑，不明白不要做，不能自己瞎做，凡变化必上报
  3. 坚持四个凡事，筑牢安全底线

    消缺经验

  4. 录波器开入开出波形异常，断续
    • 检查回路接线的各个触电是否良好稳固。
  5. 通信中断问题
    • 线路问题：查线
    • 装置问题：查接口，查通道，查配置，查设置
  6. 绝缘问题
    • 回路用表摇
    • 查看装置，例如直流电源屏的对地阻值监测

继电保护定检方法步骤

主变保护

看不同电压等级的主变保护配置，一般配置为A,B套，分别为不同厂家，也叫主一主二，另外还有后备保护，分为高中低后备，除了这些电气量的保护，还有非电气量的保护。

线路保护

安稳备自投

母线保护

母线保护配置：比率制动电流差动保护
母差基础：
首先做安措，注意关键是把外部的CT回路稳健短接，防止开路的高电压击伤操作员，然后密封同屏装置和带电端子。

验证保护逻辑，注意操作的时限和方向，比如想要母差保护动作，需要作差流，此处涉及如何加量，加量大小以及方向如何判别流入流出

验收流程

以110kV线路间隔为例，验收前要把关图纸，资料，方案等内容的审查，然后是外观——单机调试——电源——机构和回路验收——整组传动——保信录波接入——启动前验收等过程。

防跳回路验收验证（注意区分智能站和传统站）

智能站:最终看智能终端上有无跳位信号，不出口
传统站：最终传动

综自定检方法步骤

自动化验收

  1. 测控装置验收：按作业指导书进行，主要检查装置的同期功能，强合功能以及三遥数据上送是否正常。
  2. RTU远动\后台信号核对：上能量网查看信号表单、与后台对应起来、从源头做信号。
  3. 遥控功能验收：关注把手、压板是否能正确闭锁。检同期、检无压逻辑和执行条件。
  4. 远动机\后台组态配置：报备——下装——备份（有改动）

智能录波器验收

智能录波器XXXX

智能录波器配置
  1. 录波器技术规范：
    • 范围：110以上站点智能录波器的设计、施工、验收标准
    • 定义：智能录波器 intelligent recorder 一种可同时应用于智能变电站和常规变电站，集成了故障录波、网络记录分析、二次系统可视化、智能运维功能的设备，由管理单元与采集单元组成。
    • 配置要求：管理单元：110kV及以上变电站必须配置。220kV及以上变电站推荐双机配置（热备或双主工作方式）。采集单元：按电压等级和变电站规模独立配置。500kV变电站按每两串或继电器小室分散设置。
    • 功能要求：
      • 基本：信号采集（模拟量和开关量）、自检与安全（自动检测、防病毒和网络攻击）、录波召唤（按优先级）
      • 管理单元：
        • 二次系统可视化：虚回路在线监视、过程层光迁回路在线监视、二次设备状态监测
        • 智能运维管理：配置文件管控、保护设备智能巡视、二次检修辅助安措
        • 数据管理：支持录波、定值、告警文件存储和查询
      • 采集单元：
        • 支持暂态A-B时段记录和稳态记录
        • 录波通道命名规范化（如“500kV_贺罗I线_Ia”）
        • 实时解析GOOSE、SV、MMS报文。
        • 异常报文记录与告警（见附录R）。
        • 数据管理：录波文件和报文数据分区存储，支持COMTRADE格式导出。
    • 性能要求：
      • 管理单元：
        • 通信：支持调度数据网和2M方式接入，响应成功率≥95%。
        • 系统：CPU：≥4核，主频≥3.0GHz（单颗）或≥2.2GHz（多颗）。内存：≥16GB。存储：≥1TB硬盘（双硬盘配置）。
        • 数据存储：历史数据存储时间≥300天。
      • 采集单元：
        • 采样精度：交流电压回路误差≤0.5%（额定值）。交流电流回路误差≤0.5%（额定值）。
        • 记录能力：暂态记录：采样频率≥10kHz，完整记录次数≥2500次。
        • 连续报文记录：SV报文≥3天，非SV报文≥7天。
    • 通信与接口
      • 内部：
        • 管理单元通过C1与保护通信，C2与采集单元通信
        • 采集单元接入GOOSE网和站控层C2网
      • 外部：
        • 与主站通信：遵循南方电网相关通信规范，支持调度数据网和2M通道。
        • 网络安全：符合《电力监控系统安全防护规定》要求。
    • 组屏原则以及附录
  2. 根据表单测试各单元功能单体调试
  3. 同源比对功能验证联调
  4. 接入保信主站调试
  5. 南瑞继保智能录波器配置经验总结：
    1. SCD
      • 首先一定要检查，检查好配置，链路告警，注意二次回路描述，网络目录和IP地址分配，最好做好版本管理
    2. 采集单元（SUV2201A）
      • 新建工程，暂时不用管管理IP设置，选好SCD文件导入，开始配置采集单元
      • 要先订制装置，设置一下录波定值看看有没有斜率(先配置控制参数，频率10K)
      • 通道命名要合理，软件会自动适配：220kV1M_Ua/220kVXX线_Ia
      • 配置采集单元，在列表处选好物理网口（跟管理或者对下通讯），并配置好IP
      • 配置好采集单元的IEDname，对于通信很重要
      • 配置好端口配置，注意网络描述标准，网络标识，速率和历史报文改保存
      • 配置好录波配置，按通道配置好（通道、母线、线路
      • 下装进装置
    3. PCS-STATION基础组态配置
      • 导入画面填一次库
      • SCD导入采集单元，管理单元还不能（实时库还是空的
      • 修改采集的IP，导入新SCD进来
      • 填好跳闸判别点，最后发布基础组态到实时库
    4. 高级运维库
      • 更新数据，检查一二次设备情况
      • 关注交换机，可能要选出中心交换机
      • 一二次关联：把二次设备装进一次间隔（交换机要按电压等级放置
      • 一次不对应，看一下压板类型（关注差动，重合闸，停用重合闸），子类型，线路保护看看差流，智能终端看看配置管理里的硬压板
      • 二次回路可视化：看看从SCD里拉出来的物理光纤和端口（双端），网络类型。看单装置虚回路，关联链路告警（不行再更新一次，SCD对就没问题）
      • 发布一下，实时库起二次回路再配保护分析（可重启工具
      • 保护分析配置：
        • 基础模型关联采集通道前先发布采集配置（可上装）
        • 电压等级列下优先配置母线间隔（会开启拓扑）再配置其他间隔
        • 装置列下各装置点自动配置基础模型，检查遥信和遥测配置（无波形文件会无录波名，先不管），看看关联关系
        • 保存基础模型
      • 智能巡视配置
        • 厂站目录下增加定制区
        • 装置目录下去纵连通道加通道
        • 刷新配置
      • 定检
        • 刷新顶尖配置I
      • 发布高级运维组态——（同时会导出几个文件这之后就到处管理单元了
    5. 管理单元模型（IRV2201X）
      • 导出管理单元模型IRV2201X
      • 导入SCD中，最好C2网，用某个服务器IP，可配合主站
      • 对上配置，板卡目录下对上中，客户端地址设置对上IP
      • 同步管理单元里的装置
      • 再发布一次pcs_station，scd里可以导出SMCD
    6. 成图
      • 填库图里另存到厂站单线图
      • 运维组态再选项里设置好画面
      • 运维画面生成
      • 看好现场网段再对应图例，生成：过程层网络图，许回路图，平柜图，一二次不对应图
    7. 召录波文件，回去配暂态同源，完毕！
  6. 国电武仪智能录波器配置经验总结：
    1. 采集单元：通道配置+录波定值配置
      • 通道配置：订阅数据集+根据需要选择对应的电压电流通道
      • 设计并生成定值
      • 配置通信参数
    2. 管理单元：基础配置+通信配置+高级功能配置
      • 基础配置：导入SCD，查看装置各个参数，右上角全局配置
      • 通信配置：IP、IEDname、实例号、服务功能
      • 高级：从左往右，一键配置，一次间隔图关联好，画图画模板，同源比对着重挑，稳态暂态模拟量开关量都别忘了，做的途中不要忘记保存
